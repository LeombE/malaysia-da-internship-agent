# Daily Operating Protocol

## Morning

1. Check GitHub Actions latest run status.
2. Open `data/today_shortlist.csv`.
3. Apply to all realistic `A_APPLY_NOW` roles.
4. Verify and apply to strong `B_APPLY_SOON` roles.
5. Update `data/internship_tracker.csv` with applied status.
6. Note false positives and use Codex Morning Maintenance prompt to improve scoring.

## Evening

1. Review new or updated roles.
2. Prepare next-day application order.
3. Follow up on applications older than 7-10 days where appropriate.
4. Identify weak source quality or duplicates for Codex improvement.

## Weekly

1. Count applications sent.
2. Count HR replies.
3. Count interviews.
4. Count rejections.
5. Tune scoring if A bucket contains too much noise or misses good jobs.
6. Consider increasing API quota only if current source quality is proven useful.
